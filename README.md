
# 源生文档（Sourcedoc）

> 当AI开始写代码，那些困扰我们几十年的老问题，终于有了新解法。

**源生文档** 是一种与代码**同源、同步、同构**的设计上下文。它不是代码写完再补的说明书，而是和代码共享同一个起源——AI 理解需求、做出决策的推理瞬间。代码给机器执行，源生文档给人机共同体阅读。

它是 **概念宣言、标准规范、开箱即用的工具箱** —— 三位一体，为 AI 时代的软件工程铺下第一块基石。

## 为什么需要源生文档？

AI 正在成为代码的共同读者，而传统的工程实践（封装、分层、分离文档）默认为单一的人类读者优化。这导致三个永未治愈的老问题在 AI 时代被进一步放大：

- **文档同步腐烂** —— 文档与代码物理分离，永远追不上变更
- **设计意图流失** —— Why 只存在于人的脑子里，AI 和人都在黑暗中摸索
- **渐进式代码腐败** —— 每次修改都可能破坏旧设计，无人敢动旧代码

源生文档通过 **设计概要、意图文档、变更摘要、源生注释** 四层结构，将设计意图锚定在代码旁，由代码变更事件自动驱动更新，彻底改变人机协作的基础。

## 五分钟快速开始

1. 将完整规范文件 [`SOURCEDOC_SPEC.md`](./SOURCEDOC_SPEC.md) 放入项目根目录
2. 对 AI 助手说：
   > “请读取 SOURCEDOC_SPEC.md，然后帮我创建本项目的 project.overview.md”
3. 在新功能启动时，要求 AI 先创建意图文档，确认后再编码
4. 提交时要求 AI 自动输出变更摘要

完整的工作流、模板、LLM 上下文加载规则和纠错指令，请阅读全文。

## 目录结构（缩影）

```
项目根目录/
├── SOURCEDOC_SPEC.md              # 规范文件（单文件启动入口）
├── sourcedoc/                     # 源生文档目录
│   ├── project.overview.md        # 项目概览
│   ├── module/                    # 模块设计概要
│   ├── increments/                # 意图文档
│   └── change-log/                # 变更摘要
└── src/                           # 源代码（含源生注释）
```
